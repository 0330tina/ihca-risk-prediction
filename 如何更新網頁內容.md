# 🔄 如何隨時更新網頁內容

是的！您可以隨時修改網頁內容。這份指南會告訴您如何更新。

---

## ✅ 可以隨時修改

- ✅ 修改本地檔案（在您的電腦上）
- ✅ 推送到 GitHub（更新到 GitHub 上的程式碼）
- ✅ 自動部署（如果使用 Vercel 等平台，會自動更新網站）

---

## 📝 更新流程（3 個步驟）

### 步驟 1：修改檔案

1. **在 VS Code 或其他編輯器中開啟專案**
2. **修改您想要的檔案**（例如：`app/input/page.tsx`）
3. **儲存檔案**（`Ctrl + S`）

---

### 步驟 2：提交變更

在終端機中執行：

```bash
# 1. 查看修改了哪些檔案
git status

# 2. 添加所有修改的檔案
git add .

# 3. 提交變更（描述您做了什麼修改）
git commit -m "修改：更新輸入頁面樣式"
```

**提交訊息範例：**
- `"新增：添加新功能"`
- `"修改：更新按鈕顏色"`
- `"修復：修正錯誤"`
- `"優化：改善響應式設計"`

---

### 步驟 3：推送到 GitHub

```bash
git push
```

**就是這麼簡單！** 不需要 `-u origin main`，因為已經設定過了。

---

## 🚀 自動部署（如果使用 Vercel）

### Vercel 會自動更新！

如果您已經將專案部署到 Vercel：

1. **修改檔案並推送到 GitHub**
2. **Vercel 會自動偵測到變更**
3. **自動重新部署**（約 1-2 分鐘）
4. **網站會自動更新**

**您不需要做任何額外的事情！** 🎉

---

## 📋 完整範例

假設您想修改輸入頁面的標題：

### 1. 修改檔案

開啟 `app/input/page.tsx`，修改標題：

```tsx
// 從
<h1>非預期心跳驟停風險預測</h1>

// 改成
<h1>非預期心跳驟停風險預測系統</h1>
```

儲存檔案。

### 2. 在終端機執行

```bash
# 查看變更
git status

# 添加變更
git add .

# 提交
git commit -m "修改：更新輸入頁面標題"

# 推送
git push
```

### 3. 完成！

- ✅ GitHub 上的程式碼已更新
- ✅ 如果使用 Vercel，網站會自動更新（約 1-2 分鐘）

## 🔍 查看變更歷史

### 在終端機查看

```bash
# 查看提交歷史
git log

# 查看簡潔版本
git log --oneline
```

### 在 GitHub 查看

1. 前往您的 GitHub 倉庫
2. 點擊「Commits」標籤
3. 可以看到所有修改歷史

## 💡 實用技巧

### 只提交特定檔案

如果只想提交某些檔案：

```bash
# 只添加特定檔案
git add app/input/page.tsx

# 提交
git commit -m "修改：更新輸入頁面"

# 推送
git push
```

### 查看修改內容

```bash
# 查看修改了什麼
git diff

# 查看特定檔案的修改
git diff app/input/page.tsx
```

### 取消修改（如果改錯了）

```bash
# 取消所有未提交的修改（小心使用！）
git checkout .

# 或取消特定檔案
git checkout app/input/page.tsx
```

## 🆘 常見問題

### Q: 修改後網站沒有更新？

**A:** 檢查：
1. 是否執行 `git push`？
2. 如果使用 Vercel，等待 1-2 分鐘讓它自動部署
3. 重新整理瀏覽器（按 `Ctrl + F5` 強制重新整理）

### Q: 可以同時修改多個檔案嗎？

**A:** 可以！執行 `git add .` 會添加所有修改的檔案。

### Q: 修改後想回到之前的版本？

**A:** 可以！使用 `git log` 查看歷史，然後：
```bash
git checkout <commit-hash> <檔案名稱>
```

### Q: 如何知道修改了什麼？

**A:** 執行 `git status` 查看修改的檔案，`git diff` 查看具體修改內容。
